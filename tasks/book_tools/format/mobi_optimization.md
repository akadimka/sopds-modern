# План оптимизации для mobi.py

## Проблемы
1. Два класса с похожей функциональностью: Mobipocket и Mobipocket_new
2. Нарушение DRY в методах extract_cover_internal и extract_cover_memory
3. Смешение инициализации и парсинга в конструкторе
4. Отсутствие полной типизации

## Требуемые изменения
1. Удалить класс Mobipocket_new
2. Рефакторить класс Mobipocket:
   - Выделить парсинг метаданных в отдельный метод
   - Добавить ленивую инициализацию BookMobi
   - Упростить извлечение обложки
3. Добавить полную типизацию для всех методов
4. Оптимизировать работу с временными файлами
5. Добавить кэширование результатов парсинга
6. Интегрировать с единым интерфейсом парсеров
