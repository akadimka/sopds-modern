# План оптимизации для fb2sax.py

## Проблемы
1. Альтернативная реализация парсинга FB2, дублирующая функциональность fb2.py
2. Сложный ручной парсинг через xml.parsers.expat
3. Классы fb2tag и fb2cover слишком специфичны и сложны
4. Отсутствие интеграции с основной архитектурой

## Требуемые изменения
1. Определить необходимость сохранения SAX парсера
2. Если SAX парсер нужен:
   - Интегрировать его как стратегию парсинга в общую архитектуру
   - Упростить классы парсеров
   - Добавить поддержку только для извлечения обложки (где SAX эффективнее)
3. Если SAX парсер не нужен:
   - Удалить модуль полностью
   - Обновить все импорты и зависимости
4. В любом случае унифицировать интерфейс с основными парсерами FB2
5. Добавить обработку ошибок для некорректных XML файлов
