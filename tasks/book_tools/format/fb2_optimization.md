# План оптимизации для fb2.py

## Проблемы
1. Сложная иерархия наследования: FB2Base → FB2/FB2Zip
2. Дублирование функциональности с модулем fb2sax.py
3. Неоптимальное использование памяти в FB2Zip
4. Смешение парсинга и извлечения обложки

## Требуемые изменения
1. Упростить иерархию классов
2. Создать единый класс FB2Parser для парсинга метаданных
3. Оптимизировать использование памяти в FB2Zip:
   - Использовать потоковое чтение из ZIP архива
   - Избегать создания промежуточных BytesIO объектов
4. Унифицировать подход к парсингу с fb2sax.py
5. Добавить поддержку частичного парсинга для больших файлов
6. Улучшить обработку ошибок при работе с поврежденными ZIP архивами
7. Добавить кэширование результатов парсинга XML
