# Общий план оптимизации пакета book_tools.format

## Проблемы
1. Дублирование кода в классах *_new (Mobipocket_new, EPub_new, Dummy_new)
2. Нарушение принципа единой ответственности в больших классах
3. Смешение разных подходов к парсингу (lxml и SAX)
4. Неиспользуемый код в модуле aes.py
5. Неполная типизация и слабая обработка ошибок

## Требуемые изменения
1. Удалить все классы с суффиксом _new (Mobipocket_new, EPub_new, Dummy_new)
2. Создать единый интерфейс для парсеров электронных книг
3. Разделить большие классы (EPub) на компоненты с четкими ответственностями
4. Унифицировать подход к парсингу FB2 (выбрать один метод)
5. Удалить или восстановить функциональность модуля aes.py
6. Добавить полную типизацию во все модули
7. Улучшить обработку ошибок и добавить логирование
8. Создать фабрику для создания объектов книг
9. Оптимизировать использование памяти при работе с большими файлами
10. Добавить кэширование результатов парсинга

## Приоритеты
1. Высокий: Удаление дублирующихся классов, исправление mimetype.py
2. Средний: Рефакторинг больших классов, создание единого интерфейса
3. Низкий: Оптимизация производительности, добавление новых функций
