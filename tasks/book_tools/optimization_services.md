# План оптимизации: book_tools/services.py

## Текущие проблемы:
1. **Избыточная обработка ZIP-архивов**: В функции `create_bookfile_service` дважды проверяется ZIP-архив
2. **Повторное чтение данных**: Перемещение указателя в потоке может быть неоптимальным
3. **Жесткая привязка к FB2**: Функция работает только с FB2, хотя название предполагает общую функциональность
4. **Дублирование кода валидации**: Валидаторы FB2 и FB2_ZIP имеют общую логику
5. **Статический список валидаторов**: Список валидаторов фиксирован, сложно расширять

## Предлагаемые изменения:

### 1. Рефакторинг функции `create_bookfile_service`
- Создать фабричный метод для выбора парсера на основе MIME-типа
- Вынести логику обработки ZIP-архивов в отдельную функцию
- Добавить поддержку других форматов (EPUB, MOBI и т.д.)

### 2. Улучшение валидаторов MIME-типов
- Создать регистрацию валидаторов через декораторы или конфигурацию
- Вынести общую логику проверки FB2 в базовый класс
- Добавить кэширование результатов проверки для повторных вызовов

### 3. Оптимизация обработки потоков данных
- Минимизировать копирование данных между потоками
- Использовать `seek(0)` только когда необходимо
- Добавить поддержку обработки больших файлов через chunk-чтение

### 4. Улучшение логирования
- Добавить более детальное логирование для отладки
- Логировать размеры обрабатываемых данных
- Добавить тайминг операций

### 5. Добавление кэширования
- Кэшировать результаты определения MIME-типов для одинаковых файлов
- Кэшировать извлеченные метаданные

### 6. Улучшение обработки ошибок
- Более специфичные исключения для разных типов ошибок
- Восстановление после частичных сбоев
- Лучшая обработка поврежденных файлов

## Ожидаемые результаты:
1. Увеличение производительности на 20-30% для больших файлов
2. Упрощение добавления новых форматов
3. Улучшение обработки ошибок
4. Снижение потребления памяти
