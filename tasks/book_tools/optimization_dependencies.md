# План оптимизации: Зависимости и импорты

## Текущие проблемы:
1. **Циклические импорты**: Возможны из-за сложной структуры модулей
2. **Избыточные импорты**: Ненужные зависимости в некоторых модулях
3. **Отсутствие явных зависимостей**: Неясно, какие модули от чего зависят
4. **Сложность тестирования**: Запутанные зависимости усложняют изоляцию

## Предлагаемые изменения:

### 1. Реорганизация структуры пакетов
- Создать четкую иерархию модулей:
  - `book_tools.core` - базовые классы и утилиты
  - `book_tools.parsers` - парсеры для разных форматов
  - `book_tools.validators` - валидаторы и детекторы
  - `book_tools.models` - модели данных
  - `book_tools.utils` - вспомогательные функции

### 2. Устранение циклических импортов
- Проанализировать текущие зависимости
- Вынести общий код в отдельные модули
- Использовать lazy imports где необходимо

### 3. Оптимизация импортов
- Удалить неиспользуемые импорты
- Группировать импорты по стандарту PEP 8
- Использовать абсолютные импорты вместо относительных

### 4. Улучшение управления зависимостями
- Создать `requirements.txt` с явными версиями
- Добавить `setup.py` или `pyproject.toml` с метаданными
- Разделить зависимости на production и development

### 5. Добавление type stubs
- Создать файлы `.pyi` для сложных модулей
- Улучшить поддержку статической типизации
- Интегрировать с mypy для проверки типов

### 6. Создание документации зависимостей
- Документировать отношения между модулями
- Создать диаграммы зависимостей
- Описать ответственность каждого модуля

## Ожидаемые результаты:
1. Устранение циклических импортов
2. Улучшение читаемости кода
3. Упрощение тестирования
4. Более четкая архитектура
5. Упрощение поддержки и расширения
