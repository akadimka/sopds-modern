# План оптимизации: book_tools/format/

## Текущие проблемы (на основе фрагментов):
1. **Дублирование кода**: Классы `fb2cover` существуют в двух местах
2. **Устаревшие структуры**: Использование `__metaclass__ = ABCMeta`
3. **Сложная иерархия наследования**: Множественное наследование может быть избыточным
4. **Неоптимальные алгоритмы парсинга**: SAX-парсеры могут быть медленными для больших файлов
5. **Отсутствие унификации интерфейсов**: Разные классы имеют разные методы

## Предлагаемые изменения:

### 1. Консолидация дублирующегося кода
- Объединить `fb2cover` из `fb2sax.py` и `fb2parse.py`
- Создать единый модуль для работы с обложками
- Вынести общую логику в базовые классы

### 2. Модернизация структуры классов
- Заменить `__metaclass__ = ABCMeta` на современный `ABC`
- Использовать `@abstractmethod` декораторы
- Применить dataclasses для структур данных

### 3. Оптимизация парсеров
- Добавить кэширование результатов парсинга
- Реализовать ленивую загрузку метаданных
- Использовать более эффективные алгоритмы для больших файлов
- Добавить поддержку потоковой обработки

### 4. Унификация интерфейсов
- Создать единый интерфейс `BookParser` для всех форматов
- Стандартизировать методы извлечения метаданных
- Создать фабрику парсеров

### 5. Улучшение обработки обложек
- Оптимизировать извлечение и кэширование обложек
- Добавить поддержку различных форматов изображений
- Реализовать resize и оптимизацию обложек

### 6. Рефакторинг EPUB модуля
- Объединить `EPub` и `EPub_new` классы
- Устранить дублирование кода
- Улучшить обработку ошибок структуры

### 7. Добавление тестирования
- Создать модульные тесты для всех парсеров
- Добавить тесты производительности
- Создать тестовые данные для разных форматов

## Ожидаемые результаты:
1. Устранение дублирования кода
2. Улучшение производительности парсинга на 30-40%
3. Упрощение добавления новых форматов
4. Более чистый и поддерживаемый код
5. Лучшая обработка больших файлов
