# План оптимизации series_services.py

## Текущие проблемы:
1. Использование сырых SQL запросов
2. Отсутствие типизации в некоторых методах
3. Неоптимальная обработка параметров
4. Отсутствие обработки ошибок

## Требуемые изменения:

### 1. Заменить сырые SQL запросы:
- Переписать `get_series` с использованием Django ORM
- Использовать аннотации вместо сырых запросов
- Создать кастомные QuerySet методы

### 2. Улучшить типизацию:
- Добавить аннотации типов для всех методов
- Использовать TypedDict для возвращаемых значений
- Добавить проверки типов входных параметров

### 3. Оптимизировать обработку параметров:
- Валидация `lang_code` перед использованием в запросах
- Нормализация `searchterms` (приведение к верхнему регистру)
- Проверка существования `author_id` в поиске

### 4. Добавить обработку ошибок:
- Обработка `DoesNotExist` в `get_series_name`
- Валидация `searchtype` в `search_series`
- Логирование ошибок базы данных

### 5. Улучшить производительность:
- Добавить индексы на `search_ser` поле
- Использовать `prefetch_related` для связанных книг
- Кэширование часто запрашиваемых серий

### 6. Рефакторинг `search_series`:
- Разделить на отдельные методы по типам поиска
- Вынести общую логику фильтрации
- Добавить поддержку пагинации
