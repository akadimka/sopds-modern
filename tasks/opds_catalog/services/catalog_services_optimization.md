# План оптимизации catalog_services.py

## Текущие проблемы:
1. Использование глобальной переменной `DUMMY_CATALOG`
2. Отсутствие кэширования часто запрашиваемых данных
3. Неоптимальные запросы к базе данных
4. Отсутствие обработки ошибок в некоторых методах

## Требуемые изменения:

### 1. Заменить глобальную переменную:
- Создать фабричный метод для создания заглушки
- Использовать паттерн Null Object для пустого каталога

### 2. Добавить кэширование:
- Кэширование корневого каталога
- Кэширование количества книг и подкаталогов
- Кэширование пагинированных результатов

### 3. Оптимизировать запросы к БД:
- Использовать `prefetch_related` для связанных объектов
- Добавить индексы на часто фильтруемые поля
- Оптимизировать запросы подсчета

### 4. Улучшить обработку ошибок:
- Добавить обработку `MultipleObjectsReturned` для корневого каталога
- Добавить валидацию входных параметров
- Логирование всех исключений

### 5. Рефакторинг `paginated_catalog_content`:
- Разделить на более мелкие функции
- Вынести логику формирования словарей в отдельные методы
- Добавить поддержку разных стратегий сортировки

### 6. Добавить тестирование:
- Тесты для граничных случаев
- Тесты производительности для больших каталогов
