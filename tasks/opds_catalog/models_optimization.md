# План оптимизации models.py

## Текущие проблемы:
1. Отсутствие индексов на некоторых часто используемых полях
2. Нет оптимизации запросов ManyToMany через through-таблицы
3. CounterManager может создавать узкие места при частых обновлениях
4. Жестко закодированные размеры полей могут быть неоптимальны

## Требуемые изменения:

### 1. Добавить недостающие индексы:
- Индексы на поля, используемые в фильтрации и сортировке
- Составные индексы для часто используемых комбинаций полей
- Индексы для through-таблиц (bauthor, bgenre, bseries)

### 2. Оптимизировать through-таблицы:
- Добавить индексы на внешние ключи
- Рассмотреть возможность добавления полей-кэшей для счетчиков
- Оптимизировать запросы агрегации через through-таблицы

### 3. Улучшить CounterManager:
- Добавить кэширование счетчиков
- Реализовать batch-обновление счетчиков
- Добавить асинхронное обновление счетчиков
- Ввести механизм отложенного обновления для редко меняющихся счетчиков

### 4. Оптимизировать размеры полей:
- Проанализировать фактическое использование размеров полей
- Рассмотреть возможность использования TextField для annotation
- Оптимизировать хранение search_* полей

### 5. Добавить методы-помощники:
- Методы для получения связанных объектов с prefetch
- Методы для агрегации данных
- Методы для batch-операций

### 6. Улучшить обработку lang_code:
- Добавить валидацию lang_code
- Создать методы для фильтрации по языковым кодам
- Оптимизировать запросы с группировкой по lang_code

### 7. Добавить миграции для оптимизации:
- Миграции для добавления индексов
- Миграции для изменения типов полей при необходимости
- Миграции для добавления denormalized полей для оптимизации

### 8. Оптимизировать запросы:
- Добавить аннотации для часто запрашиваемых агрегатов
- Использовать select_related/prefetch_related в менеджерах
- Создать кастомные QuerySet для часто используемых запросов
